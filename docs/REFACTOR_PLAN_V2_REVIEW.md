## 重要更正：Union 引用类型变体支持

### 验证测试

**测试代码**（`/tmp/test_union_ref.uya`）：
```uya
union IntOrRef {
    i: i32,
    r: &i32,  // 引用类型变体
}

fn test_union_ref() void {
    var x: i32 = 42;
    const u: IntOrRef = IntOrRef.r(&x);
    match u {
        .i(v) => { },
        .r(v) => { },
    };
}
```

**编译结果**：✅ 成功

**生成的 C 代码**：
```c
union IntOrRef {
    int32_t i;
    int32_t * r;  // 引用类型变体完全合法！
};
struct uya_tagged_IntOrRef { int _tag; union IntOrRef u; };
```

**结论**：Uya union **完全支持引用类型变体**（`&T`）。

### 阶段四：Union 数据结构重构 ✅ 完全可行

**重要更正**：经深度代码分析和实际测试，Uya union **完全支持引用类型变体**。

#### 4.1 Type 结构体 Union 化

**原方案**（验证可行）：
```uya
union TypeData {
    void_unit: void,           // TYPE_VOID, TYPE_I8-I64, TYPE_BOOL 等
    named: &byte,              // TYPE_ENUM, TYPE_STRUCT, TYPE_UNION 的名称
    pointer: PointerData,      // TYPE_POINTER（结构体变体）
    array: ArrayData,          // TYPE_ARRAY（结构体变体）
    slice: SliceData,          // TYPE_SLICE（结构体变体）
    tuple: TupleData,          // TYPE_TUPLE（结构体变体）
    error_union: ErrorUnionData, // TYPE_ERROR_UNION（结构体变体）
    generic_param: &byte,      // TYPE_GENERIC_PARAM（引用变体）
}

struct PointerData {
    pointee: &Type,  // ✅ 引用类型字段
    is_ffi: i32,
}
```

**可行性分析**：
- `named: &byte` → 引用类型变体 → **完全支持**
- `pointer: PointerData` → 结构体变体（内含 `&Type` 字段）→ **完全支持**
- 所有变体类型都符合 Uya union 规范

**工作量评估**：5 天
